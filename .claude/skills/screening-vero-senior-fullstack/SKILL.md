# Position-Specific Screening: vero-senior-fullstack

**Generated from calibration review on 2026-04-13. 10 profiles reviewed with hiring manager.**

## How to Use This File

Read this BEFORE screening any candidate for vero-senior-fullstack. Apply these rules IN ADDITION TO the general screening skill. When these rules conflict with the general screening skill, THESE RULES WIN.

## Calibrated Must-Haves (user-validated)

1. **5+ years of RELEVANT experience at real tech companies** — count only years at actual tech/product/SaaS companies. Food companies, parking apps, agriculture, hardware companies do NOT count. If someone has 2 years at a good tech company and 6 years at non-tech, they do NOT meet the 5+ year requirement.
2. **At least one of: Go, Python, Node.js, TypeScript** verified in skills or work history
3. **Backend or fullstack** — must have real backend experience. Pure frontend developers are rejected even with "Software Engineer" title.
4. **Israel-based, on-site in Tel Aviv area**
5. **Startup or strong tech company experience** (up to 500 employees)

## Calibrated Dealbreakers (user-validated)

- Standard Israeli tech dealbreakers: consulting/outsourcing as ONLY experience (Tikal, Matrix, Ness, Develeap, Sela, Taldor, Sapiens, Amdocs)
- Banks, government, defense, telcos as primary experience
- Companies over 500 employees as ONLY experience (need startup DNA)
- Pure frontend developers (React/CSS/HTML only, no backend)
- Outside Israel
- **HOWEVER: consulting/outsourcing in career history is OK** if the candidate also has strong tech company experience or matching cybersecurity/AI skills. This is different from other positions where consulting is a hard dealbreaker.

## Title Rules

**QUALIFY:**
- Senior Software Engineer
- Senior Backend Developer / Engineer
- Senior Full Stack Developer / Engineer
- Software Engineer (if 5+ years)
- Backend Developer (if senior enough)

**REJECT:**
- Team Lead / Tech Lead / Engineering Manager — too senior, role is hands-on IC
- Frontend Developer / Frontend Engineer / UI Developer — wrong focus
- DevOps Engineer / SRE — wrong function
- Data Scientist / ML Engineer — wrong function (unless also strong in backend dev)
- Junior / Intern / Associate — too junior
- CTO / VP Engineering / Director — overkill for IC role

## Stack Verification

**Must have at least ONE of:**
- Go (Golang) — strongest signal for this role
- Python — strong signal
- Node.js — good signal
- TypeScript — good signal (often paired with Node.js)

**Acceptable but not sufficient alone:**
- Scala, Java, C++ — strong backend languages but not the required stack. QUALIFY if the company is strong (e.g., Wix backend with Scala = OK because Wix is a strong signal)
- React.js, Vue.js — only acceptable if PAIRED with backend skills (Node.js, Python, etc.)

**Reject if:**
- Only Java/Spring with no modern stack
- Only .NET/C# (legacy)
- Only frontend technologies (React, HTML, CSS, Angular)

## Relevant Experience Duration

**This was the #1 rejection pattern in calibration.**

Count ONLY years at real tech companies. Reject if:
- Total relevant experience < 5 years even if total career is 10+ years
- Only the last role is at a tech company with < 4 years there
- Previous companies are food, agriculture, parking, healthcare, hardware

- Sapir Malatzky: Only last role at accessiBe (~3yr). Before: John Bryce (training). REJECTED.

## Key Takeaway for This Position

**The hiring manager values RELEVANT EXPERIENCE DEPTH over total career length.** A candidate with 15 years but only 2 at a real tech company is worse than someone with 6 years all at startups. Cybersecurity background is a strong differentiator. Consulting is OK if skills match (unlike other positions). Team Lead is too senior — want hands-on ICs. Always verify backend skills from actual profile, not title.
